# Git Merge Conflict

## 1. বর্তমান কনফ্লিক্ট অবস্থা থেকে বের হয়ে আসুন (যেহেতু আগের মার্জ ফেইল করেছে):
```bash
git merge --abort
```


## 2. নিচের কমান্ডটি দিয়ে নতুন করে মার্জ করুন (যেখানে dev_rahul-এর কোড প্রাধান্য পাবে):
```bash
git merge -X theirs dev_rahul
```
এই কমান্ডটি কী করবে?   

- -X theirs (এক্সটেন্ডেড স্ট্রাটেজি অপশন): এটি শুধুমাত্র যেখানে কনফ্লিক্ট (দুই দিকের কোডে অমিল) হয়েছে, সেখানে শুধুমাত্র dev_rahul ব্রাঞ্চের কোডটি নিয়ে নেয়। যে লাইনে কোনো কনফ্লিক্ট নেই, সেগুলো স্বয়ংক্রিয়ভাবে মার্জ হয়ে যাবে ।

- ফলাফল: আপনার লোকাল ব্রাঞ্চের যেকোনো পরিবর্তন (যেখানে কনফ্লিক্ট ছিল) ওভাররাইট হয়ে যাবে এবং dev_rahul এর কোড বসে যাবে। আপনার লোকাল ব্রাঞ্চে যেই কোড ছিল যেটা dev_rahul-এ নেই, সেগুলো ডিলিট হয়ে যাবে না, বরং ঠিক থাকবে (শুধু কনফ্লিক্ট এলাকা ওভাররাইট হবে) ।

## ⚠️ সাবধানতা: "Ours" এবং "Theirs" বুঝে ব্যবহার করুন
আপনি "incoming accept" চাচ্ছেন, মানে আপনি dev_rahul-কে প্রাধান্য দিতে চাচ্ছেন। সেক্ষেত্রে নিচের টেবিলটি দেখে নিন:

| আপনি যদি চান                         | কমান্ড                            | ব্যাখ্যা                                    |
| ------------------------------------- |:--------------------------------:|:----------------------------------------:|
| `dev_rahul` (Incoming) জিতুক          | `git merge -X theirs dev_rahul`  | কনফ্লিক্ট জায়গায় dev_rahul-এর কোড নিবে।   |
| আপনার লোকাল কোড (Current) জিতুক   | `git merge -X ours dev_rahul`    | কনফ্লিক্ট জায়গায় আপনার বিদ্যমান কোড রাখবে। |

** দয়া করে নোট করুন:** `git checkout --theirs` এর মতো কমান্ডের সাথে এটি গুলিয়ে ফেলবেন না। 
`checkout --theirs` পুরো ফাইলটিই রিপ্লেস করে দেয়, কিন্তু `merge -X theirs` শুধু কনফ্লিক্ট অংশটুকু রিপ্লেস করে ।

## যদি ভুল করে ফেলেন?
`git reset --hard` করার পর যদি বুঝতে পারেন যে পরিবর্তনগুলো দরকার ছিল:
```bash
# শেষ 5 টি দেখুন
git reflog

# একটি নির্দিষ্ট অবস্থায় ফিরে যান
git reset --hard HEAD@{2}    # reflog থেকে নম্বর দিবেন
```




# আপনার করা সব পরিবর্তন মুছে ফেলতেচান এবং শেষ কমিটের অবস্থায় ফিরিয়ে আনতে চান

```bash
git branch --show-current
```

আপনি যদি git pull নেওয়ার আগে আপনার লোকাল পরিবর্তনগুলো পুরোপুরি বাদ দিয়ে 
রিমোটের সাথে মিলিয়ে নিতে চান, তাহলে নিচের কমান্ডগুলো ব্যবহার করুন:
## Current Branch

## 1. আপনার সব লোকাল পরিবর্তন বাদ দিতে:
```bash
git reset --hard HEAD
```
এটি আপনার করা সব পরিবর্তন মুছে ফেলবে এবং শেষ কমিটের অবস্থায় ফিরিয়ে নিবে।

## 2. দ্রুত সমাধান (এক লাইনে):
```bash
git reset --hard HEAD && git pull
```
অথবা আরও সরাসরি:
```bash
git fetch origin && git reset --hard origin/your-branch-name
```

## 3. তারপর রিমোট থেকে আপডেট নিতে:
```bash
git pull
```

# বিভিন্ন পরিস্থিতির জন্য সমাধান:

## পরিস্থিতি ১: শুধু unstaged পরিবর্তন (modified but not added)
```bash
git restore .                    # সব ফাইল রিস্টোর করবে
git clean -fd                    # নতুন ফোল্ডার/ফাইল ডিলিট করবে
git pull
```

## পরিস্থিতি ২: staged পরিবর্তনও আছে (git add করা)
```bash
git reset --hard HEAD            # সব পরিবর্তন হারাবে
git pull
```

## পরিস্থিতি ৩: কমিট করে ফেলেছেন, কিন্তু পুশ করেননি
```bash
git reset --hard origin/your-branch-name    # রিমোটের সাথে মিলিয়ে নিবে
```

## পরিস্থিতি ৪: স্ট্যাশ করে রাখতে চান (পরে লাগতে পারে)
```bash
git stash                         # পরিবর্তনগুলো স্ট্যাশ করে রাখে
git pull                          # ক্লিন স্টেটে পুল নেয়
git stash drop                    # স্ট্যাশ ডিলিট করে (যদি না চান)
```
